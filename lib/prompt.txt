If you're already planning an **interactive pre-quiz questionnaire**, then I would actually remove some of the profile fields and move them into the quiz creation flow.

### Keep in Profile (Long-term Preferences)

These rarely change:

* Class / Grade
* Age (optional)
* Study Goal
* Interests
* Learning Topics
* Preferred Language
* Difficulty Preference
* Study Hours/Week
* Target Exam
* Learning Style

### One More Suggestion

Don't show 13+ questions at once.

Instead use a conversational wizard:

```text
AI: What topic would you like to study today?

User: Quantum Mechanics

AI: Great. Which areas should I focus on?
□ Wave Functions
□ Schrödinger Equation
□ Operators
□ Mixed

User: Mixed

AI: How challenging should the quiz be?
○ Easy
○ Medium
○ Hard
○ Adaptive

AI: How many questions?

User: 20
```

Then auto-fill everything else from the user's profile.

Most users will abandon a setup flow if they have to answer 15+ questions every time. A good rule is:

* **Profile = permanent preferences**
* **Quiz wizard = session-specific preferences**
* **Analytics = inferred automatically from performance**

That usually gives the best balance between personalization and completion rate.
